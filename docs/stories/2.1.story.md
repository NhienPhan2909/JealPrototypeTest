# Story 2.1: Public Site Layout & Home Page

## Status

**Done**

## Story

**As a** car buyer,
**I want** to land on a welcoming home page with clear navigation to browse vehicles,
**so that** I can quickly understand what the dealership offers and start shopping.

## Acceptance Criteria

1. React application initialized in `/frontend` folder with Create React App or Vite
2. React Router v6 installed and configured with basic routing structure
3. App layout component created with header (dealership name/logo, navigation links) and footer (basic copyright text)
4. Home page route `/` displays hero section with dealership name, tagline, and prominent "Browse Inventory" call-to-action button
5. Navigation header includes links: "Home", "Inventory", "About"
6. "Browse Inventory" button links to `/inventory` route
7. Dealership name and logo fetched from API (`GET /api/dealers/:id`) and displayed dynamically in header
8. Basic CSS styling applied (Tailwind CSS or plain CSS) for clean, professional appearance
9. App runs locally with `npm start` and displays home page without errors
10. Dealership ID is configurable (e.g., via URL parameter, environment variable, or hardcoded for prototype) so public site knows which dealership to display

### Tech Stack Requirements

**Frontend Framework:** React 18.2+ with Vite 5.0+ build tool

**Rationale:** Vite dev server starts in <1 second vs CRA's 10-30 seconds. HMR is instant. Minimal config. CRA is deprecated in favor of Next.js, but Next.js is overkill for SPA.

**CSS Framework:** Tailwind CSS 3.4+

**Rationale:** Utility-first = fast iteration, no context switching to CSS files, well-documented, zero runtime JS.

**Routing:** React Router 6.20+

**State Management:** React Context + useState for global state (dealership selection)

**HTTP Client:** Native fetch API (zero dependencies, adequate for simple GET/POST/PUT/DELETE)

**Key Decision - JavaScript over TypeScript:**
Plain JS with JSDoc comments for critical functions is faster for 2-day MVP. TypeScript deferred to Phase 2.

### Layout & Header Component Patterns

**Layout.jsx Responsibilities:**
- Render Header component
- Render page content via `<Outlet />` (React Router)
- Render Footer component
- Provide consistent wrapper for all public pages

**Header.jsx Responsibilities:**
- Fetch dealership info from `/api/dealers/:id` on mount
- Display dealership name and logo (if exists)
- Render navigation links: Home, Inventory, About
- Use React Router `<Link>` components for client-side navigation
- Handle loading and error states

**Example Structure:**

```javascript
import { Link } from 'react-router-dom';
import { useState, useEffect } from 'react';

function Header() {
  const [dealership, setDealership] = useState(null);
  const [loading, setLoading] = useState(true);
  const dealershipId = 1; // Hardcoded for MVP

  useEffect(() => {
    fetch(`/api/dealers/${dealershipId}`)
      .then(res => res.json())
      .then(data => {
        setDealership(data);
        setLoading(false);
      })
      .catch(err => {
        console.error('Failed to load dealership:', err);
        setLoading(false);
      });
  }, []);

  if (loading) return <div>Loading...</div>;

  return (
    <header className="bg-white shadow">
      <div className="container mx-auto px-4 py-4 flex justify-between items-center">
        <div className="flex items-center gap-4">
          {dealership?.logo_url && (
            <img src={dealership.logo_url} alt={dealership.name} className="h-12" />
          )}
          <h1 className="text-2xl font-bold">{dealership?.name}</h1>
        </div>
        <nav className="flex gap-6">
          <Link to="/" className="hover:text-blue-600">Home</Link>
          <Link to="/inventory" className="hover:text-blue-600">Inventory</Link>
          <Link to="/about" className="hover:text-blue-600">About</Link>
        </nav>
      </div>
    </header>
  );
}

export default Header;
```

### Home Page Component Pattern

**Home.jsx Responsibilities:**
- Fetch dealership info from `/api/dealers/:id` on mount
- Display hero section with dealership name as H1 heading
- Display tagline (from `about` field or generic text)
- Render "Browse Inventory" CTA button linking to `/inventory`
- Handle loading state while fetching data

**Example Structure:**

```javascript
import { useState, useEffect } from 'react';
import { Link } from 'react-router-dom';

function Home() {
  const [dealership, setDealership] = useState(null);
  const [loading, setLoading] = useState(true);
  const dealershipId = 1; // Hardcoded for MVP

  useEffect(() => {
    fetch(`/api/dealers/${dealershipId}`)
      .then(res => res.json())
      .then(data => {
        setDealership(data);
        setLoading(false);
      })
      .catch(err => {
        console.error('Failed to load dealership:', err);
        setLoading(false);
      });
  }, []);

  if (loading) return <div className="text-center py-12">Loading...</div>;

  return (
    <div className="bg-gradient-to-r from-blue-500 to-blue-700 text-white py-20">
      <div className="container mx-auto px-4 text-center">
        <h1 className="text-5xl font-bold mb-4">{dealership?.name}</h1>
        <p className="text-xl mb-8">
          {dealership?.about?.substring(0, 100) || 'Quality vehicles at great prices'}
        </p>
        <Link to="/inventory" className="btn-primary text-lg px-8 py-3">
          Browse Inventory
        </Link>
      </div>
    </div>
  );
}

export default Home;
```

### API Integration

**Dealership API Endpoint:**
- **GET /api/dealers/:id** - Fetch single dealership details

**Request Example:**
```javascript
const response = await fetch('/api/dealers/1');
const dealership = await response.json();
```

**Response Format:**
```json
{
  "id": 1,
  "name": "Acme Auto Sales",
  "logo_url": "https://res.cloudinary.com/...",
  "address": "123 Main St, Springfield, IL 62701",
  "phone": "(555) 123-4567",
  "email": "sales@acmeauto.com",
  "hours": "Mon-Fri 9am-6pm, Sat 10am-4pm, Sun Closed",
  "about": "Family-owned dealership serving Springfield for 20 years.",
  "created_at": "2025-11-19T10:00:00.000Z"
}
```

**Error Handling:**
- 404: Dealership not found → Display error message
- 500: Server error → Display generic error message
- Network error: Display "Unable to load data" message
