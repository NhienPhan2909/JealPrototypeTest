12. No JavaScript errors in browser console for any page or user interaction

### Mobile Touch Target Sizing

**WCAG 2.1 AA Requirement:** Minimum 44x44 CSS pixels for touch targets

**How to Ensure 44px Height:**
- For buttons: Use `py-3` instead of `py-2` to increase vertical padding
- For input fields: Use `py-3` to increase input height
- For links: Ensure adequate line-height or padding

**Tailwind Class Examples:**
```javascript
// Button with 44px+ height
<button className="bg-blue-600 text-white px-4 py-3 rounded hover:bg-blue-700">
  Submit
</button>

// Input field with 44px+ height
<input className="border border-gray-300 rounded px-3 py-3 w-full" />

// Link with adequate tap target (via padding)
<a href="#" className="text-blue-600 hover:underline inline-block py-2 px-2">
  Click here
</a>
```

**Testing Touch Target Size:**
- Use browser DevTools to inspect element dimensions
- Measure height: should be >= 44px
- Test on actual mobile device: can you easily tap without mis-clicking?

- Optimize images: reduce size, use Cloudinary transformations (e.g., `w_800,f_auto,q_auto`)
- Defer non-critical JavaScript
- Remove unused CSS/JS (unlikely with Vite's tree-shaking)

### Responsive Design Best Practices

**Mobile-First Approach:**
1. Write base styles for mobile (no breakpoint prefix)
2. Add `md:` prefix for tablet overrides (>= 768px)
3. Add `lg:` prefix for desktop overrides (>= 1024px)

**Example:**
```javascript
// Mobile: single column, smaller text, full-width button
// Tablet: two columns, medium text, auto-width button
// Desktop: three columns, large text, auto-width button
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <h2 className="text-2xl md:text-3xl lg:text-4xl">Title</h2>
  <button className="w-full md:w-auto btn-primary">Submit</button>
</div>
```

**Common Responsive Issues and Fixes:**

**Issue:** Horizontal scrolling on mobile
- **Cause:** Fixed-width elements, images too large, negative margins
- **Fix:** Use `max-w-full` on images, `w-full` on containers, remove fixed widths

**Issue:** Text too small on mobile
- **Cause:** Font size < 16px, user must zoom to read
- **Fix:** Use Tailwind default `text-base` (16px) or larger for body text

**Issue:** Buttons too small to tap
- **Cause:** Button height < 44px
- **Fix:** Increase vertical padding: `py-2` ‚Üí `py-3`

**Issue:** Grid layout doesn't stack on mobile
- **Cause:** Missing `grid-cols-1` base class
- **Fix:** Add `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` pattern

**Issue:** Navigation menu doesn't collapse on mobile
- **Cause:** No mobile menu implementation (hamburger icon, hidden nav)
- **Fix:** Implement mobile menu with `hidden md:flex` pattern or hamburger icon

```javascript
/**
 * Header - Public site navigation with mobile hamburger menu.
 */
function Header() {
  const [mobileMenuOpen, setMobileMenuOpen] = useState(false);

  return (
    <header className="bg-white shadow-md">
      <div className="max-w-6xl mx-auto px-4 py-4 flex justify-between items-center">
        {/* Logo/Brand */}
        <div className="text-2xl font-bold">Dealership Name</div>

        {/* Desktop Navigation (hidden on mobile) */}
        <nav className="hidden md:flex space-x-6">
          <Link to="/" className="text-gray-700 hover:text-blue-600">Home</Link>
          <Link to="/inventory" className="text-gray-700 hover:text-blue-600">Inventory</Link>
          <Link to="/about" className="text-gray-700 hover:text-blue-600">About</Link>
        </nav>

        {/* Mobile Hamburger Icon (visible on mobile only) */}
        <button
          className="md:hidden text-gray-700 focus:outline-none"
          onClick={() => setMobileMenuOpen(!mobileMenuOpen)}
          aria-label="Toggle menu"
        >
          {/* Hamburger icon (3 lines) */}
          <svg className="w-6 h-6" fill="none" stroke="currentColor" viewBox="0 0 24 24">
            <path strokeLinecap="round" strokeLinejoin="round" strokeWidth={2} d="M4 6h16M4 12h16M4 18h16" />
          </svg>
        </button>
      </div>

      {/* Mobile Navigation Menu (shown when mobileMenuOpen is true) */}
      {mobileMenuOpen && (
        <nav className="md:hidden bg-gray-50 px-4 py-2">
          <Link to="/" className="block py-2 text-gray-700 hover:text-blue-600" onClick={() => setMobileMenuOpen(false)}>
            Home
          </Link>
          <Link to="/inventory" className="block py-2 text-gray-700 hover:text-blue-600" onClick={() => setMobileMenuOpen(false)}>
            Inventory
          </Link>
          <Link to="/about" className="block py-2 text-gray-700 hover:text-blue-600" onClick={() => setMobileMenuOpen(false)}>
            About
          </Link>
        </nav>
      )}
    </header>
  );
}
```

**Key Patterns:**
- Desktop nav: `hidden md:flex` (hidden on mobile, flex on tablet+)
- Mobile hamburger: `md:hidden` (visible on mobile, hidden on tablet+)
- Mobile menu: Conditionally rendered based on `mobileMenuOpen` state
- Close menu on link click: `onClick={() => setMobileMenuOpen(false)}`

### Console Error Checking

**How to Check Console for Errors (AC: 12):**

**Chrome DevTools:**
1. Open DevTools: F12 or Ctrl+Shift+I (Windows) or Cmd+Opt+I (Mac)
2. Click "Console" tab
3. Look for red error messages
4. Common error types:
   - `Uncaught TypeError`: JavaScript type errors (e.g., calling method on undefined)
   - `Failed to load resource`: Network errors (e.g., 404, 500 API errors)
   - `Warning`: React warnings (not critical but should be fixed)

**Firefox Browser Console:**
1. Open Browser Console: Ctrl+Shift+K (Windows) or Cmd+Shift+K (Mac)

**Transformation Examples:**
